1. 数组是值类型

在golang中，基本数据类型和数组都是值类型

什么是值类型？

定义：
当一个变量赋值给另一个变量时，会复制值本身，两个变量互不影响

也就是说，每个变量都有自己独立的数据副本

如：

a := 10
b := a  // 复制了 a 的值

b = 20  // 改变 b，不会影响 a

fmt.Println("a =", a)
fmt.Println("b =", b)

对数组，也有这个结论

1. Array（数组）的介绍

数组是指一系列同一类型数据的集合。

数组的类型由 **基本数据类型 和 长度 **定义

2. 数组的定义

fmt.Printf("arr1:%T,arr2:%T,str1:%T", arr1, arr2, str1)

注意，数组的长度也是类型的一部分

数组的初始化

var arr1 [3]int

fmt.Println(arr1)

定义了一个[3]int的数组，直接打印，得到的结果为[0 0 0]

和基本数据类型保持一直，如果是int类型，结果是0，如果是string类型，结果是空

1. 方法1：一一赋值

var arr1 [3]int

arr1[0] = 1

arr1[1] = 10

arr1[2] = 24

fmt.Println(arr1)

结果为[1 10 24]

在golang中，数组长度指定之后，长度便不可再变

2. 方法2：定义和赋值一起

var arr1 = [3]int{1, 10, 24}

fmt.Println(arr1)

结果为[1 10 24]

3. 方法3：类型推导方式

arr1 := [3]int{1, 10, 24}

fmt.Println(arr1)

结果为[1 10 24]

4. 方法4：让编译器自行推断数组长度

var arr1 = [...]int{1, 10, 24}

fmt.Println(arr1)

利用[…]让编译器自行推断数组的长度

打印的结果为[1 10 24]

不能用于增加arr1的长度：

打印数组长度len()

var arr1 = [...]int{1, 10, 24}

fmt.Println(len(arr1))

输出结果为3

5. 方法5：注明索引值的初始化

arr1 := [...]int{0: 1, 1: 10, 2: 20, 5: 50}//在5处赋值50

fmt.Println(len(arr1))

在0处赋值1，在1处赋值10，在2处赋值20，在5处赋值50

该代码的结果为6，表名这个数组中存储了6个int类型数字

2. switch case

switch case语句可方便对大量的值进行条件判断

练习：判断文件类型，如果后缀名是.html，输入text/html,如果后缀名是.css 输出text/css ……

语法：

switch expression{

case condition：

}

当然，我们首先可以想到if来判断：

var ext = ".html"

if ext == ".html" {

​ fmt.Println("text/html")

} else if ext == ".css" {

​ fmt.Println("text/css")

} else if ext == ".js" {

​ fmt.Println("text/javascript")

} else {

​ fmt.Println("找不到此后缀")

}

用switch case实现上面功能：

var ext = ".html"

switch ext {

case ".html":

​ fmt.Println("text/html")

​ break

case ".css":

​ fmt.Println("text/css")

​ break

case ".js":

​ fmt.Println("text/javascript")

​ break

default:

​ fmt.Println("找不到此后缀")

}

注意，case condition的后面一定接上冒号“:” swtich变量后面紧接着花括号”{}”，在golang中，break可以写也可以不写，其他每行都要在任务执行完加default

switch case的另一种写法

将定义的变量作为局部变量放在swtich代码的里面，需要加分号分割：

switch ext := ".html";ext {

case ".html":

​ fmt.Println("text/html")

​ break

case ".css":

​ fmt.Println("text/css")

​ break

case ".js":

​ fmt.Println("text/javascript")

​ break

default:

​ fmt.Println("找不到此后缀")

}

依然可以正常运行

判断一个数是不是偶数

switch case同时给判断赋多个值：

var num = 5

switch num {

case 1, 3, 5, 7, 9:

​ fmt.Println("这是奇数")

case 2, 4, 6, 8:

​ fmt.Println("这是偶数")

这里我们给每种case都赋值多个值，每个值之间用逗号连接，依然可以正常运行

判断年龄

在switch上可以写表达式：

var age = 30

switch { //不接名称

case age < 20:

​ fmt.Println("年轻人")

case 20 < age && age < 60:

​ fmt.Println("中年人")

default:

​ fmt.Println("老年人")

}

注意，如果接的是表达式，此时switch后面不接变量名称。需要用判断时，建议用if else语句

switch的穿透 fallthrought

fallthrought语法可以执行满足条件的case的下一个case：

var age = 30

switch {

case age < 20:

​ fmt.Println("年轻人")

case 20 < age && age < 60:

​ fmt.Println("中年人")

​ fallthrough

case age > 60:

​ fmt.Println("老年人")

default:

​ fmt.Println("输入错误")

}

该代码本来只会执行 中年人，但是有fallthrought，将下一行的结果 老年人也执行了。注意，一个fallthrought只能穿透一层，也就是只会多执行紧挨着的下面一层的结果。

1.byte和rune类型

组成每个字符串的元素叫”字符“，可以通过便利字符串元素获得字符，字符用单引号(‘’)包裹

如何定义一个字符

字符属于int类型

var a ='a' 这么定义是正确的

var a ='aa' 这么定义是错误的，byte类型只能含有一个元素

var a = 'a'

fmt.Printf("类型是%T,字符是%v", a, a)

得到的结果是类型是int32,字符是97

可见，默认输出的结果是a对应的ASCII码的值，类型是int32

如何原样输出a?

使用%c来原样输出a：

var a = ‘a’

fmt.Printf(“类型是%T,字符是%c”, a, a)

Go 语言的字符有以下两种：

  1. uint8 类型，或者叫 byte 型，代表了 ASCII 码的一个字符。
  2. rune 类型，代表一个 UTF-8 字符。

定义一个字符串，输出字符串里面的字符：

var str = "This"

fmt.Printf("值：%v,类型：%T", str[1], str[1])

取到了字符h，对应105。在golang中，一个字母(ASCII)占用一个字节的空间，一个汉字(UTF-8)占用三个字节的空间

查看字符串中占用存储空间

前面介绍过unsafe,sizeof查看int，float类型等的大小，但是对于字符串来说，这是行不通的，无论我们的字符串有多长，都只会返回16：

在 Go 里，字符串 (string) 本质上并不是直接存储字符数组，而是一个 结构体，它包含两个字段：

type stringStruct struct {
str unsafe.Pointer // 指向底层字节数组的指针
len int // 字符串长度
}

  • str 占 8 字节（64 位机器上指针大小 = 8）
  • len 占 8 字节（64 位机器上 int = 8）

所以总共就是 16 字节。

想获取字符串占用大小，可以用len直接获取字节数，这对于汉字也是可以用的，比如：

var a = "数据"

fmt.Println(len(a))

返回的值为6

用字符定义一个汉字

var a = '数'

fmt.Printf("值：%v,类型：%T", a, a)

得到的输出为 25968,类型：int32

因为golang中使用UTF-8来进行编码，在UTF-8中编码后使用的就是该值

通过循环遍历字符串里的字符

package main

import (
“fmt”
)

func main() {
s := “hello中国”

// 1. len -> 字节数
fmt.Println("len(s):", len(s)) // 11 (hello 5 + 中国 6)

// 2. []byte -> 按字节切分
fmt.Println("[]byte(s):", []byte(s))

// 3. []rune -> 按 Unicode 码点切分
fmt.Println("[]rune(s):", []rune(s))
fmt.Println("len([]rune(s)):", len([]rune(s)))

// 4. range -> 按 rune 遍历
for i, r := range s {
	fmt.Printf("index=%d, rune=%c, codepoint=%d\n", i, r, r)
}

}

index=2, rune=l, codepoint=108
index=3, rune=l, codepoint=108

注意，rune的功能更强大，因为ASCII码是UTF-8的子集

range和len的区别

在用for循环打印字符串中的汉字时，打印出的根本不是其对应的UTF-8编码，为了彻底搞清楚这个问题，查阅了很多资料：

首先是go的字符串储存的方式：Go 的字符串底层 统一用 UTF-8 编码存储

为什么用range能正常打印出汉字呢？

因为range 数的是字符，而len数的是字节，以下是他们的区别：

操作 返回/处理对象 行为
len(s) 字节数 返回 UTF-8 编码后占用多少字节，不涉及字符概念
s[i] 第 i 个字节 按字节取值，如果直接打印成 %c，可能显示“乱码”
range s rune（字符/码点） 按 UTF-8 解码，逐个返回完整的字符（Unicode 码点）

此外，如果直接打印字符的值，得到的是一个五位数，而循环打印，得到的是三个三位数，这是为什么？

比如：

s := "中"

s1 := '中'

fmt.Printf("%v\n", s1)

for i := 0; i < len(s); i++ {

​ fmt.Printf("%v ", s[i])

}

得到的输出是：

20013
228 184 173

打印方式 访问对象 输出 为什么
[]rune(s) 或 range Unicode 码点 20013（五位数） rune 表示完整字符的码点值
按字节循环 s[i] UTF-8 字节 228, 184, 173（三位数） UTF-8 把一个字符编码成多个字节，每个字节单独打印就是单个整数

每个字符都有一个唯一的 Unicode 码点，例如 "中" 的码点是 20013（十进制）。
在 UTF-8 编码下，它占用 3 个字节 [228 184 173]（其他编码方式可能不同）。
当使用 range 或转换成 []rune 遍历字符串时，程序会把 UTF-8 字节序列解码成对应的码点，再用 %c 打印时显示对应字符。
而如果用按字节循环（如 for i := 0; i < len(s); i++）逐个打印，就只能看到 UTF-8 的原始字节值 [228 184 173]，所以看起来像“三个三位数”

一句话而言：range是按字符，而len是按字节

修改字符串

要修改字符串，需要先将其转换成[]rune 或[]byte，完成后再转换为 string。无论哪种转换，
都会重新分配内存，并复制字节数组。

我们想要将big中的b改为p，想到用下面的方法

s1 :="big"

s1[0] = 'p'

fmt.Println(s1)

然而结果会直接报错，这是因为GO语言规定：s1 是 string 类型，本质上是一个不可变的字节序列

这里有一些关于byte的小常识：

byte只能处理单个字节，byte的本质是0-255的整数，是ASCII码的别名， 也就是当我们定义或者取字符的时候，实际上定义或者取的是0-255的一个数，所以只能接收字符常量或整数值，不能直接转换整个字符串。如果要处理字符串，要么取单个索引（s[0]），要么转成 []byte

a := byte('a')

fmt.Println(a)

b := byte(255)

fmt.Println(b)

以上两种是可以直接用byte转化的例子，得到的结果为

97
255

我们想修改字符串，可以定义切片，修改切片，最后转化为string类型再输出，例如：

str := "big"

strChange := []byte(str)

strChange[0] = 'p'

fmt.Println(string(strChange))

这里我们有常识：切片里面的每个数字都对应一个字符，可以通过string来转化为字符串

比如这个代码，它转化过程就是先把big转化为切片，取切片的第一个元素，用一个新的字符更改，最后再转化为字符串输出，如果最后一行没有string转换，输出的是[112 105 103]。

如果字符串中含有汉字呢？

我们使用rune来转换：

str := "你 好,big"

strChange := []rune(str)

strChange[0] = '大'

strChange[1] = '家'

fmt.Println(string(strChange))

最后得到的输出为 大家好,big

当然，如果全为字母，也可以直接使用rune转换，所以说rune比byte功能更强大
